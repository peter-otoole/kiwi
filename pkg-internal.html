Description

Documentation for the internals of Arrow. Every sufficiently large utility library will contain its own utility library. These functions are required to reduce repetition of code when implementing arrow functions.

stringify_call

stringify_call is a tool for formatting the source call of an error message. It fixes as issue with printing ':=' functions, and cuts off very long calls.

colourise

colourise is a set of functions that wraps strings in ansii escape sequences on coloured terminals, allowing coloured text to be printed. These are used to colour error messages and print methods.

write_error

write_error prints a red error message to the terminal, if the terminal supports colour.

assert

assert is a key arrow function - it takes an expression, a call to display, and a string message. If the expression isn't true, the error is thrown with that message and call.

insist

insist is a list of functions that provide a minimal interface to an assertion. This object exists to reduce the amount of assertion checking code needed.

insist $ must_be_fn_matchable(fn, invoking_call)

Each function encloses a message function to stop the function being repeatedly created when assertions are ran. When the assertion inside the insist function fails, an error is thrown.

demand

An internal object containing assertions that are required by the collection comprehension functions. Mostly contains parse error assertions.

one_of

Return one value from a collection.

equals

R's equal operator doesn't work on lists or strange values. equals is a better measure of identity.

call_with_params

Construct a call to a function 'fnname' with the parametres of a second function. Useful for higher order functions.

+

Concatenate two strings.

in

An infix function to test for the non-membership of an element in a set.

Object

Construct an empty environment.

join_env

Join two environments together into one environment. This allows for inheritance of environments without having to traverse multiple environments.

is_fn_matchable

Is a value a function, or possibly the name of a function.

is_collection

Is a value a generic or atomic vector or a pairlist.
